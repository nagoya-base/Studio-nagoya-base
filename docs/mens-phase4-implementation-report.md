## 4. 代替判断した項目（Phase3指定アセットが未制作のもの）

指示書5章の判断順に従い、以下は「Phase3指定のファイルが実在しないため、既存アセットで代替する」第2優先の判断を維持した。理由をここに明記する。

### 4-1. ヒーロー背景画像

* **Phase3の記載**：`gallery_man02_1200.webp` を元に、ヒーロー用に横長トリミングした新規ファイル `hero_mens_1600.webp`／`.jpeg` を書き出す（Phase3自身も「本フェーズでは画像加工を行わない」と明記し、トリミング作業をPhase4実装者に委ねている）。
* **未制作の理由**：本修正で変更を許可されたファイルは `mens/index.html`・`styles/mens.css`・本報告書の3点のみであり（指示書11章）、`images/` への新規ファイル追加は許可範囲外。また実際の画像トリミング・書き出しには画像編集作業が必要で、本セッションでは実施しなかった。
* **採用した代替案**：Phase3が第1候補に指定した元画像 `gallery_man02_1200.webp` を、トリミング済みファイルを介さずCSS背景（`background-size: cover` 相当の `.hero` 実装）として直接使用。表示結果としてはブラウザ側で自動的に画面比率に合わせてクロップされるため、視覚的な意図（男性モデル・麻縄・宙吊りが分かるヒーロー背景）はPhase3の第1候補と一致する。

### 4-2. OGP画像（`og:image` / `twitter:image` / `LocalBusiness.image`）

* **Phase3の記載**：新規ファイル `ogp-mens-v2-1200x630.jpg` を、ヒーロー候補画像とコピー・スタジオ名・地名のテキスト合成によって新規制作する（8章）。「この合成・書き出し作業はPhase4以降（画像実制作の指示が出た段階）で行う」と明記されている。
* **未制作の理由**：4-1と同様、画像の新規合成はファイル変更許可範囲外であり、実施していない。
* **採用した代替案**：
  * `og:image` / `twitter:image` は、指示書6章が明示的に許容する「専用OGP画像の仕様のみ確定・実画像未制作の場合は既存の中立的なスタジオ内観画像を暫定使用してよい」という規定に従い、現行の `ogp_IMG_2309_1200.jpeg`（人物なし・一般トップページと共有の既存画像）を維持した。ポートレート比率の男性モデル写真をそのままSNSカードに使うとプラットフォーム側の自動クロップで顔や結び目が欠ける懸念があるため、既存の横長比率（1200×675）画像を暫定的に優先した。
  * `LocalBusiness.image`（schema.orgの代表画像。SNSカードのような自動クロップの影響を受けにくい）は、Phase3が新OGP画像の合成元に指定した `gallery_man02_1200.webp` を直接使用した。Phase3 19-3が求める「男性向けページであることをより明確に示す」方針を、専用ファイルが用意できるまでの間、最も近い形で満たすための判断。

## 8. ヒーロー背景の差し戻し

Phase 4初版・Phase 3再照合修正では、ヒーロー背景をPhase 3が第1候補とする男性モデル写真（`gallery_man02_1200.webp`）に差し替えていたが、その後ユーザーからの直接の指示（「トップのヒーロー画像は元の部屋の写真に戻して欲しい」）により、一般トップページと共通の部屋の写真（`IMG_2309_1600.webp`）へ差し戻した。

* `mens/index.html`：`<section class="hero mens-hero" ...>` から `mens-hero` クラスを削除し、preloadタグを `IMG_2309_1600.webp` に変更
* `styles/mens.css`：不要になった `.hero.mens-hero` の背景画像上書きルールを削除

これによりヒーロー背景は共通CSS（`styles/common.css`）のデフォルト値にフォールバックし、一般トップページと同一の表示になる。ギャラリー内の `gallery_man02_1200.webp`（吊り撮影の作例）自体は変更していない。7章「残課題」からヒーロー専用トリミング画像（`hero_mens_1600.*`）の項目を削除したのは、ヒーロー背景が男性モデル写真の使用自体を取りやめたため、当該ファイルが不要になったことによる。

## 9. OGP画像の確定

## 11. 最終改善指示：利用人数／作例方針の確定

前回指示書にあった「男性1名利用の強調」「男性ペア作例の必須追加」に関する記述を、以下の方針に差し替える。

### 11-1. 利用人数の基本方針

当スタジオは1〜4名で利用可能であり、この一文が1名利用・2名利用（モデル＋撮影者）・緊縛の縛り手と受け手・3名以上での吊り利用・男性ペアや男性グループでの撮影や練習をすべて包含している。特定の利用人数や構成を過度に強調する必要はなく、ヒーロー・特徴・FAQへ「男性1名でも利用できます」を積極的に追記することはしない。既存の定員表示（`hero-badges` の「最大4名」、料金セクションの利用時間の目安、規約上の1〜4名表示）で足りるものとし、本改訂ではこれらの表示を変更していない。

### 11-2. ソロ作例の扱い

男性モデルが1人だけ写った作例（`gallery_man01`・`gallery_man02`・`gallery_man03`・`hero_mens_1200`）は、いずれも正式な男性向け作例として扱う。実際の利用人数と完成写真に写る人数は一致しない。2〜3名（男性モデル・カメラマン・縛り手・安全管理者等）で利用した場合でも、完成写真には男性モデル1名のみが写る構図は一般的にあり得るため、ソロ作例であることを理由に男性同士の利用文脈を裏付けられないとは判断しない。

### 11-3. 男性ペア作例に関する方針

男性2人が同時に写った写真は必須作例ではない。旧版の7章で残課題としていた「男性ペア・男性グループが同時に写った写真が0点」は、本改訂により残課題から削除した（7章参照）。自然で適切な男性ペア作例が将来撮影できた場合は追加してよいが、現時点での必須対応とはしない。

### 11-4. ギャラリー説明文

`#gallery` セクションの説明文を、写真に写る人数の言及から、男性モデルの撮影用途の説明へ変更した。

* 変更前：「男性同士の緊縛、吊り撮影、ユニフォーム、フェティッシュ衣装などの作例です。」
* 変更後：「男性モデルによる緊縛、吊り撮影、ユニフォーム、コスチューム撮影の作例です。」

実際に男性2人が写っていない写真を「ペア作例」と説明する表現は使用していない。

### 11-5. ページ上部の対象者説明・特徴・FAQ

「このページは、男性同士で緊縛やフェティッシュ撮影を楽しみたい方に向けたご案内です。」（`#about-page`）、特徴「男性同士で使える」（`#features`）、FAQ「男性同士で利用できますか？」（回答：「はい。男性ペア・男性グループでの緊縛練習、撮影、作品撮りにご利用いただけます。利用できるのは18歳以上の方に限り、高校生は利用できません。」）は、いずれも現行の文言をそのまま維持し、本改訂による変更は行っていない。「男性同士」は写真へ同時に2人写ることではなく、利用者・モデル・撮影者・縛り手などを含む利用文脈を示す語として扱う。1名利用の可否は定員・料金表示（1〜4名）で判断できるため、FAQへの追記は行わない。

* 男性モデルのソロ作例が正式な作例として扱われている：達成済み（11-2章）
* 完成写真に写る人数と実際の利用人数を混同していない：達成済み（11-2〜11-4章、ギャラリー説明文を撮影用途の説明に変更）
