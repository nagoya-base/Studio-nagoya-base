# /mens/ 改修 Phase 2：構成・導線設計提案書

対象ページ：`mens/index.html`（公開URL: https://nagoya-base.github.io/Studio-nagoya-base/mens/）

本書はPhase 2の成果物であり、**ページ構成・情報の順序・重複要素・CTA配置の設計のみ**を目的とする。
`mens/index.html` 本体、CSS、JS、画像は一切変更していない。コミット・プッシュ・公開も行っていない。
コピー（文言）はPhase 1（`docs/mens-phase1-copy-proposal.md`）で確定した内容をそのまま使用し、本書では新規の文言確定は行わない。

---

## 1. ページの目的

`/mens/` は、一般トップページと**別のサービスを提供するページではない**。
一般トップページと同じレンタルスペース（Studio Nagoya Base）について、男性同士の緊縛練習・フェティッシュ撮影を検討している利用者が、心理的な距離を感じずに予約まで進める**男性向け専用入口**として設計する。

料金・設備・予約方法・安全ルールはトップページと共通化し、差分は「対象者の明示」「男性作例」「訴求文」に限定する。

---

## 2. 想定利用者

* 男性同士で緊縛練習や緊縛撮影をしたい人
* ゲイ・バイ男性を含む男性ペア・男性グループ
* ユニフォーム、スポーツウェア、制服、レザー等のフェティッシュ撮影をしたい人
* 女性モデル中心の緊縛作例に心理的な距離を感じる人
* 完全個室・無人入室のレンタルスペースを探している人

### 現状の主な問題点

| 問題 | 内容 |
|---|---|
| 対象者訴求とレンタルスペース訴求の分離 | ヒーロー・案内文で「男性同士」「レンタルスペースであること」をまとめて説明するセクションが存在しない |
| 講習会訴求の残存 | 特徴・利用用途の両方に「講習会」表現が残り、Phase 1で削除確定した内容と矛盾している |

| 独立した安全ルールセクションがない | 安全条件はFAQ・特徴・利用規約に分散しており、規約を読む前に要点を把握できる場所がない |

| ギャラリーaltが汎用的 | `Studio Nagoya Base gallery image 1` 等、内容を説明しないalt文が残っている（Phase 1 13章で修正方針は確定済み、実施はPhase 3） |
| ニュース欄が一般ページと共通 | 男性向け利用に関係しない一般告知も表示される可能性がある |
| ファンサイト撮影が主要用途に混在 | 利用用途セクションの主要項目に「ファンサイト・有償コンテンツ撮影」が並び、主訴求（緊縛・フェティッシュ）と並列に見える |

## 8. CTA配置

現行は6箇所以上（ヒーロー／カレンダー直後／CTA二分岐／特徴末尾／設備末尾／料金末尾）に「空き状況を確認」系ボタンが存在し、押し売り感が出ている。

改修後は以下の**最大4箇所**に絞る。

| 配置 | 役割 |
|---|---|
| ① ヒーロー | 最初の行動喚起（空き確認・問い合わせ） |
| ② カレンダー直後（予約CTA二分岐） | カレンダーを見た直後の予約／相談の分岐 |
| ③ ギャラリーまたは特徴の直後 | 内容を理解した利用者への中間喚起 |
| ④ ページ最下部（最終CTA） | 読了後の最終喚起 |

同じ文言・同じボタンを短い間隔で連続配置しない。設備セクション末尾・料金セクション末尾のCTAは③または④に統合し、単独では置かない。

同じ情報を複数ファイルへ直接複製する構成は避け、将来内容がずれる可能性があるものは共通includeへ寄せる方針とする（安全ルール・利用の流れが該当）。

## 13. FAQの順序

指示書5-12に基づき、以下の順序を採用する。対象者理解に関わる項目を上位、料金・運用系を下位に配置する。

1. 男性同士で利用できますか
2. ゲイ・バイ男性でも利用できますか（新規）
3. 緊縛師やモデルを手配してもらえますか（Phase 1で新規追加確定済み）
4. 縛り方を教えてもらえますか
5. 緊縛・吊り撮影は可能ですか
6. フェティッシュ衣装で撮影できますか（Phase 1で新規追加確定済み）
7. ユニフォームやスポーツウェアで撮影できますか
8. ファンサイト向け撮影は可能ですか
9. 性的な撮影は可能ですか（新規・7と重複しないよう「性的行為・性的行為を伴う撮影の禁止」に特化した回答とする）
10. スマホで撮影できますか
11. 機材を持ち込めますか
12. 支払い方法
13. キャンセルポリシー
14. 飲食・アルコール
15. 駐車場
16. 忘れ物

新規FAQ（2・9）の回答文案：

```
Q. ゲイ・バイ男性でも利用できますか？
A. はい。ゲイ・バイ男性を含む男性ペア、男性グループでの利用を歓迎しています。性別や性的指向による利用制限はありません。

Q. 性的な撮影は可能ですか？
A. 衣装や身体表現を含む作品撮りは可能ですが、性的行為および性的行為を伴う撮影は禁止しています。
```

FAQ本文と特徴セクションで同じ文章を繰り返さない（特徴＝短い価値訴求、FAQ＝条件の詳細）。構造化データ（`FAQPage.mainEntity`）は画面表示の並び・文言と一致させる。

## 14. ギャラリー分類

### カテゴリ

* 緊縛・床縄
* 吊り撮影
* ユニフォーム
* スポーツウェア
* フェティッシュ衣装
* カラーライティング
* 白背景・身体表現

現行の「白ホリ・カラーバリエーション」「フルカラー照明」「和の背景」という表示区分を、上記7カテゴリに再編する。

### 写真方針

* 女性モデルの写真を使用しない
* 男性モデル中心で統一する
* 性的行為を連想させる写真は避ける
* 設備や利用シーンが理解できる写真を優先する
* 裸体のみの写真に偏らない
* 衣装・照明・吊り設備の幅を見せる
* 同じ構図・同じ衣装の写真を連続させない

### 枚数

最低6枚、推奨8〜12枚。質の低い写真を埋め合わせで追加しない。

## 16. 実装時の注意事項（Phase 3向け）

1. 「縛られる体験をしてみたい方へ」セクションの削除は `mens/index.html` 内のみに閉じた変更であることを確認済み（一般トップページ `index.html` には同セクションが存在しない）。ただし、外部リンク先（`ataru-nagoya` サイト）から `/mens/` への逆リンクが存在するかは別途確認が必要。
2. 安全ルールセクションは既存の共通include `_includes/phase2_trust_ja.html` の `#safety` ブロックを流用する。指示書が挙げる「吊り利用は3名以上」「首を使った吊りは禁止」等の項目が同includeに含まれない場合、共通include側に追加するか、メンズページ側で補足するかをPhase 3で判断する（共通パーツの内容変更は一般トップページにも影響するため、変更前に一般ページ側への影響を確認する）。
3. 「利用の流れ」は現状ページごとに個別記述されている。共通include化する場合、一般トップページの表示文言も含めて調整が必要になるため、影響範囲を確認してから着手する。
4. ギャラリーalt文の書き換え、画像の追加・差し替え、OGP画像の制作はいずれもPhase 3の作業であり、Phase 2では行わない。
5. 構造化データ（`LocalBusiness.description` / `FAQPage.mainEntity`）は、Phase 1で確定した文言・本書で確定したFAQ順序と完全に一致させる。画面に存在しない説明を構造化データのみに追加しない。
6. アクセシビリティ要件（Phase 3実装時に遵守）：
   * セクション見出しの順序を正しく保ち、H1は1つのみとする
   * H2・H3の階層を崩さない
   * CTA文言だけで遷移先が分かるようにし、「こちら」「詳しくはこちら」単独のリンクテキストを避ける
   * ギャラリー画像には内容を説明するaltを付け、装飾画像は空altを検討する
   * FAQはボタン操作・キーボード操作に対応する（`<details>`要素は現状維持で対応可能）
   * 文字色と背景色のコントラストを確保する
   * センシティブな写真をページ読み込み直後に過度に大きく表示しない
7. 料金・予約方法・利用規約・トップページ本体は本Phaseでも次のPhaseでも変更しない（指示書3-2の前提を維持）。
8. 会員料金の申告制運用（「メンズページを見た」と伝える）は現行のまま維持し、独自の料金表・予約フォームは新設しない。
