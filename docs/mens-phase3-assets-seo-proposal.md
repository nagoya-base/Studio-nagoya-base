# /mens/ 改修 Phase 3：写真・OGP・SEO設計提案書

対象ページ：`mens/index.html`（公開URL: https://nagoya-base.github.io/Studio-nagoya-base/mens/）

本書はPhase 3の成果物であり、**使用画像・OGP・SEO情報の仕様確定のみ**を目的とする。
`mens/index.html` 本体、CSS、JS、画像ファイルは一切変更・削除・変換していない。画像の実制作（OGP新規撮影・加工）も行っていない。

> **前提資料についての注記**：指示書は `docs/mens-phase2-structure-proposal.md` を前提資料としているが、本リポジトリには当該ファイルが存在しない（Phase 2の構成提案書は未作成、または未コミット）。そのため本書は `docs/mens-phase1-copy-proposal.md`（コピー確定案）と、現行 `mens/index.html` の実装、一般トップページ `index.html`、`images/` フォルダの実物確認のみを根拠に作成した。Phase 2文書が別途確定した場合、本書の見出し構造・ギャラリー掲載順（7章）は整合性を再確認すること。

## 1. Phase 3の目的

Phase 1・Phase 2で確定したページ方針（男性同士の緊縛・フェティッシュ撮影に使える完全個室レンタルスペース）に基づき、以下を確定する。

- 使用する既存画像の棚卸しと採用・不採用の判定
- ヒーロー画像・ギャラリー構成
- OGP画像の仕様
- 画像ファイル名・サイズ・形式・alt文
- title / meta description / OGP / Twitter Card / canonical
- 構造化データ（LocalBusiness / FAQPage）の修正内容
- SNS共有確認項目
- Phase 4実装者への変更一覧

写真と検索結果から「男性向け」「男性同士の緊縛・フェティッシュ対応」「レンタルスペース（緊縛サービスではない）」「完全個室・無人入室」「名古屋・上前津」「吊り床・カラー照明・撮影設備」が一目で伝わる状態を目指す。

## 2. 写真の基本方針

指示書3章の方針をそのまま採用する。

- **女性モデルが写った緊縛写真は使用しない**（設備単体写真は女性モデルが写っていなければ使用可）
- 人物入り作例は**男性モデル中心**で統一する
- 性的行為を直接想起させる構図・性器が見える写真・過度な露出・苦痛や暴力性のみが強調された写真は使用しない
- モデルの性的指向を写真・alt文・キャプションで断定しない（「ゲイ男性」等の断定表現は不使用）
- 緊縛写真だけに偏らせず、衣装撮影・照明作品・設備写真も含めて「レンタルスペースとしての用途の幅」を伝える

この方針に照らし、3章で棚卸しした結果、**現行 `images/` フォルダには女性モデルの緊縛写真が4点存在し、これらはすべて `/mens/` では不採用**と判定した（4章参照）。

### 棚卸しから判明した制約（重要）

## 8. OGP仕様

### 8-1. 目的・構成・注意事項

指示書10章の方針をそのまま採用する。

- 伝えるべき内容：男性向けページ／緊縛・フェティッシュ用途／完全個室レンタルスペース／名古屋・上前津
- 推奨コピー：「男性同士の緊縛・フェティッシュ撮影を、完全個室で。」
- 補助文字：「Studio Nagoya Base」「名古屋・上前津」
- 女性モデルを使用しない、性器・過度な露出を含めない、レンタルスペースであることが分かる構成にする、文字を端に寄せすぎない

### 8-2. 現行OGP画像の問題点

現行 `og:image`（`ogp_IMG_2309_1200.jpeg`）は、

1. サイズが1200×675であり、OGP標準の1200×630と異なる（9章で確認済み）
2. 男性モデルも緊縛・カラー照明の要素も写っていない、スタジオ内観のみの写真である
3. **一般トップページ（`index.html`）と全く同じファイルを共有しており**、`/mens/` 用に差し替えると一般トップページ側のOGPには影響しないが、ファイル自体を上書き・削除すると一般トップページのOGPが壊れる
